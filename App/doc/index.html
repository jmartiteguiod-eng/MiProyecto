Class Cita POJO Model

Estructura de datos unificada para mapear de manera directa y bidireccional los documentos NoSQL de la colección raíz desnormalizada user_citas en Firebase Cloud Firestore.

Campos e Instancias de Clase

Modificador y TipoCampoDescripción
private StringfechaFecha programada para la reserva. Formato estándar persistido: yyyy-MM-dd.
private StringhoraHora exacta de inicio de la prestación del servicio. Formato persistido: HH:mm.
private StringservicioDenominación comercial del servicio contratado (ej. "Corte de pelo").
private StringuserIdUID global generado e indexado por Firebase Auth correspondiente al cliente solicitante.
private StringestadoCiclo de vida interno de la reserva. Valores de estado: "realizada", "no_realizada" o null.

Class TimeSlot POJO Model

Clase encargada de definir el modelo elemental de una franja de tiempo fija de 30 minutos dentro de la agenda diaria comercial del negocio.

Constructores

Constructor y Definición técnico-arquitectónica
public TimeSlot()
Constructor vacío explícito. Requisito técnico imprescindible exigido por el SDK nativo de Firebase para permitir la desreferenciación automática del modelo cloud mediante la función document.toObject(TimeSlot.class) sin lanzar excepciones en tiempo de ejecución.
public TimeSlot(String time, boolean available, boolean isPast)
Constructor parametrizado de sobrecarga utilizado en tiempo de renderizado para inicializar la celda horaria calculando su elegibilidad basándose en la marca temporal actual del sistema.

Class TimeSlotAdapter RecyclerView.Adapter

Controlador y adaptador encargado de inyectar el listado dinámico de franjas horarias y maquetar la lógica visual del selector de turnos en base a su disponibilidad.

Métodos Críticos de Interfaz

Modificador y TipoMétodo y Documentación Funcional
public VHonCreateViewHolder(ViewGroup parent, int viewType)
Instancia la estructura física de la vista inflándola a partir del recurso XML interno item_time_slot.xml y vincula el objeto con el contenedor estático de memoria.
public voidonBindViewHolder(VH holder, int position)
Aplica las reglas de negocio estéticas sobre el botón interactivo según el estado de la cita: si la hora ya ha expirado o está reservada, inhabilita el clic táctil y reduce el canal alfa (opacidad visual) entre el 35% y 40%. Si está disponible, vincula los escuchadores de eventos para procesar la selección del usuario.

Class CitasActivity Activity Component

Controlador gráfico del lado del cliente encargado de recuperar, listar secuencialmente y gestionar las acciones relativas a las reservas vigentes asociadas al usuario autenticado.

Lógica y Flujo de Procesamiento

Modificador y TipoMétodo e Impacto en Base de Datos
protected voidonCreate(Bundle savedInstanceState)
Punto de entrada de la actividad. Vincula las referencias del ListView y del botón de cancelación, inicializa los componentes del SDK de Firestore y gatilla el proceso de sincronización asíncrona.
private voidloadUserCitas()
Lanza una petición estructurada (Query) a Firestore apuntando a la colección raíz y aplicando una cláusula condicional de filtrado (WhereEqualTo) con el UID del usuario logueado, poblando dinámicamente el adaptador gráfico en tiempo real.
private voidmostrarDialogoCancelacion(int posicion)
Despliega un cuadro de diálogo flotante (AlertDialog) de carácter seguro. Si el usuario confirma, realiza una llamada a Firestore borrando o actualizando el documento seleccionado y sincronizando la interfaz de usuario en el acto.

Class ReservarCitaActivity Activity Component

El componente neurálgico del flujo de reservas de la aplicación. Maneja la lógica de negocio síncrona, validación previa de conectividad a la red y el control transaccional concurrente de agendas.

Métodos y Lógica de Negocio

Modificador y TipoMétodo y Control de Concurrencia NoSQL
private voidcheckDisponibilidadFecha(String fecha)
Accede a la colección en la nube, descarga el estado de los bloques de turnos ocupados para la fecha seleccionada y cruza dinámicamente los datos en local para invalidar los horarios duplicados en la pantalla de reservas.
private voidconfirmReservation(View vistaBoton)
Validación final de seguridad. Inspecciona que los campos de selección requeridos no sean nulos y efectúa una inserción transaccional atómica en la base de datos cloud. En caso de éxito, despliega una alerta guiada (Snackbar) y redirige el flujo de control.
